Package Tracking App

Help people to track all packages at one place

Timeline

3 Days

Skills
  • UX Design
  • Visual Design
  • Prototyping

Background

This was a side project I quickly made for exploring mobile interactions by using the new prototyping tool, ProtoPie, so my focus was on UI and interaction in this project. I designed three main screens: Homepage, Add Sheet, Camara Scan, which showcased the main feature of this app.


The key feature of this app is to help people tracking all their packages in one place, so users can add the package to this app by scanning or sync their carrier or shopping account to this app. Also, this app will remind the user if there’s anything they need to be aware of, for example, signiture required.

Icons

Since shipping status has a few stages, I thought it would be interesting to see some micro-interaction when the shipping status gets updated. My initial idea was to use the shape transformation to animate one icon to another icon, so I picked the flat style.


Progress Bar

I explored three color combinations on the progress bar. The first version is the blue-gray tone. I think this color is great for the background illustrations but too subtle to show the interaction. In the end, I picked the brown tone instead of blue because it’s the color related to “package”.


When shipping status gets updated, the icon on the progress bar will change and move forward. The number of days left will be recalculated as well.


Filter & Accordion Cards

The users are not only want to track packages that are sending to them but also packages they sent out, such as return items, products they sold on eBay. Hence, I think to add a filter on this app can help them quickly tracking the package they are looking for.


For arrive-today packages, my initial idea is to use the carousel to display multiple cards, but I found it will be more convenient if I can see all packages’ status in the same viewport. That’s why I decide to use accordion formate here.
